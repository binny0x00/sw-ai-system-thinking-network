- 네트워크 응용프로그램은 동일한 기본 프로그래밍 모델인 `클라이언트-서버(client-server) 모델`에 기초한다.

### 11.1 클라이언트-서버 프로그래밍 모델

#### 핵심 개념

- `클라이언트(client)`는 서비스를 요청하는 프로세스다.
- `서버(server)`는 자원(resource)을 관리하면서 클라이언트에게 서비스를 제공하는 프로세스다.
- 클라이언트와 서버는 물리적 컴퓨터가 아니라 **프로세스의 역할**이다.
  - 하나의 호스트에서 여러 클라이언트·서버 프로세스가 동시에 실행될 수 있다.
  - 하나의 프로세스가 상황에 따라 두 역할을 모두 수행할 수도 있다. (예: 웹 서버가 API 서버에 요청을 보낼 때는 클라이언트)

#### 용어 정리

| 용어 | 설명 |
|------|------|
| `프로세스(process)` | 운영체제가 관리하는 실행 중인 프로그램 인스턴스 |
| `호스트(host)` | 네트워크에 연결되어 IP 주소 같은 식별자를 갖는 컴퓨터 |
| `자원(resource)` | 서버가 관리하는 파일, 데이터, 계산 결과 등 서비스 대상 |
| `트랜잭션(transaction)` | 클라이언트 요청 → 서버 응답의 한 요청-응답 사이클 (DB의 ACID 트랜잭션과 다른 의미) |

> **Host vs Machine**: Machine은 물리적 컴퓨터, Host는 네트워크에 연결된 컴퓨터. 관계: `Machine(Host) ⊃ Process`

#### 동작 방식

![figure 11.1 클라이언트-서버 트랜잭션](assets/figure_11_1.png)

1. 클라이언트가 서버에 `요청(request)`을 전송한다.
2. 서버는 요청을 수신·해석하고 필요한 자원에 접근하거나 연산을 수행한다.
3. 서버가 결과를 `응답(response)`으로 만들어 클라이언트에 반환한다.
4. 이 사이클이 반복되면서 네트워크 응용프로그램이 동작한다.

- 서버는 보통 클라이언트 요청을 `listen` 상태(대기)로 기다리고, 클라이언트는 필요할 때만 연결해 요청한다.
- 클라이언트와 서버는 반드시 다른 호스트일 필요 없다. (예: `localhost` 서버와 브라우저 통신)

#### 예시

- 웹 브라우저(클라이언트)가 특정 URL을 요청하면, 웹 서버(Apache, Nginx 등)가 해당 파일이나 동적 결과를 응답하는 것이 하나의 트랜잭션이다.
